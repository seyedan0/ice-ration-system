# Database Schema — Ice Ration Distribution System
## Engine: PostgreSQL 16+

---

## Entity Relationship Overview

```
users ──< roles
stations ──< users (agents)
stations ──< daily_tickets ──< citizens
citizens ──< daily_tickets
daily_tickets ──< ticket_claims
stations ──< inventory_logs
deliveries >── stations
deliveries >── users (drivers)
```

### `daily_tickets`
One record per citizen per calendar day. Generated by the nightly cron job.

| Column | Type | Constraints | Notes |
|---|---|---|---|
| `id` | BIGSERIAL | PK | |
| `citizen_id` | BIGINT | FK → citizens(id) NOT NULL | |
| `station_id` | BIGINT | FK → stations(id) NOT NULL | Station where citizen is assigned for that day |
| `ticket_date` | DATE | NOT NULL | The calendar day this ticket is valid for |
| `allocated_blocks` | SMALLINT | NOT NULL | Copied from citizen.daily_ration at generation time |
| `status` | ENUM | NOT NULL DEFAULT 'pending' | `pending`, `claimed`, `expired` |
| `claimed_at` | TIMESTAMPTZ | NULL | Set when status → claimed |
| `claimed_by_agent_id` | BIGINT | FK → users(id) NULL | Which agent confirmed the claim |
| `created_at` | TIMESTAMPTZ | DEFAULT NOW() | |

**Indexes:**
- `UNIQUE (citizen_id, ticket_date)` — enforces one ticket per citizen per day
- `(station_id, ticket_date, status)` — for agent dashboard queries
- `ticket_date`, `status`

### `inventory_logs`

| Column | Type | Constraints | Notes |

## Key Constraints Summary

```sql
-- One ticket per citizen per day
ALTER TABLE daily_tickets ADD CONSTRAINT uq_citizen_date UNIQUE (citizen_id, ticket_date);

-- Station stock cannot go negative
ALTER TABLE stations ADD CONSTRAINT chk_stock_non_negative CHECK (current_stock >= 0);

-- Allocated blocks must be positive
ALTER TABLE daily_tickets ADD CONSTRAINT chk_blocks_positive CHECK (allocated_blocks > 0);
ALTER TABLE citizens ADD CONSTRAINT chk_ration_positive CHECK (daily_ration > 0);
```

## Citizen Lookup Query (used in every validation call)

```sql
SELECT c.*, dt.id AS ticket_id, dt.status, dt.allocated_blocks, dt.claimed_at
FROM citizens c
LEFT JOIN daily_tickets dt
  ON dt.citizen_id = c.id AND dt.ticket_date = CURRENT_DATE
WHERE
  c.national_id = $1
  OR c.mobile = $1
  OR c.qr_code = $1
LIMIT 1;
```

This single query covers all three identifier types.

## Claim Ticket (Atomic Transaction)

```sql
BEGIN;

-- Lock the ticket row
SELECT id, status, allocated_blocks
FROM daily_tickets
WHERE citizen_id = $citizen_id AND ticket_date = CURRENT_DATE
FOR UPDATE;

-- Verify still pending (application layer checks this too)
UPDATE daily_tickets
SET status = 'claimed', claimed_at = NOW(), claimed_by_agent_id = $agent_id
WHERE id = $ticket_id AND status = 'pending';

-- Deduct from station stock (CHECK constraint prevents going negative)
UPDATE stations
SET current_stock = current_stock - $allocated_blocks,
    updated_at = NOW()
WHERE id = $station_id;

-- Write audit log
INSERT INTO inventory_logs (station_id, change_type, blocks_delta, stock_after, reference_id, performed_by)
VALUES ($station_id, 'ration_out', -$allocated_blocks,
        (SELECT current_stock FROM stations WHERE id = $station_id),
        $ticket_id, $agent_id);

COMMIT;
```
